# 飲食店向けシフト管理システム

## 概要

飲食店のシフト作成業務を支援するWebアプリケーションです。

従業員情報、勤務希望、必要人数、定休日、臨時休業を管理し、必要人数に対して出勤希望人数が足りているかを確認できます。
不足している日や時間帯をカレンダーや一覧で可視化することで、シフト作成時の確認作業を支援します。

## 作成背景

前職の飲食店勤務では、従業員ごとの勤務希望、時間帯ごとの必要人数、扶養内勤務、勤務時間の上限など、シフト作成時に考慮する条件が多くありました。

特に、時間帯ごとの人手不足を確認する作業や、曜日ごとの必要人数を毎回入力する作業に負担を感じたため、勤務希望と必要人数を管理し、不足時間を可視化できるシステムを作成しました。

## 主な機能

### 従業員管理

* 従業員の登録
* 従業員の一覧表示
* 従業員情報の編集
* 従業員の削除
* 入力チェック

登録できる項目は以下のとおりです。

* 氏名
* 雇用形態
* 熟練度
* 時給
* 月上限時間
* 月上限金額

### 勤務希望管理

* 勤務希望の登録
* 勤務希望の一覧表示
* 勤務希望の編集
* 勤務希望の削除
* 開始時刻と終了時刻の入力チェック
* 同一従業員・同一日付・重複時間帯の登録防止

登録できる項目は以下のとおりです。

* 従業員
* 日付
* 開始時刻
* 終了時刻
* 希望区分

### 日付別必要人数設定

* 日付・時間帯ごとの必要人数登録
* 必要人数の一覧表示
* 必要人数の編集
* 必要人数の削除
* 開始時刻と終了時刻の入力チェック
* 必要人数が1人以上であることの入力チェック
* 同一日付・重複時間帯の登録防止

### 必要人数パターン設定

曜日ごとの基本的な必要人数パターンを登録できます。

* 曜日パターンの登録
* 複数曜日への一括登録
* 曜日パターンの一覧表示
* 曜日パターンの編集
* 曜日パターンの削除
* 同一曜日・重複時間帯の登録防止

例：

* 月〜金 09:00〜11:00 2人
* 土日 09:00〜11:00 3人

### 月別反映機能

登録済みの曜日パターンを、指定した年月の日付別必要人数へ反映できます。

すでに同じ日付・重なる時間帯の必要人数が登録されている場合は、上書きせずにスキップします。
また、定休日や臨時休業日に設定されている日は、必要人数を作成しないようにしています。

### 定休日設定

毎週の定休日を設定できます。

例：

* 毎週水曜日を定休日にする
* 毎週月曜日を定休日にする

定休日は不足状況カレンダーで「定休日」として表示されます。

### 臨時休業設定

特定の日だけ休業する日を登録できます。

登録できる項目は以下のとおりです。

* 休業日
* 理由

例：

* 2026-07-10 祭り
* 2026-08-13 お盆休み
* 2026-12-31 年末休業

臨時休業日は、不足状況カレンダーで「臨時休業」として表示されます。

### 不足状況カレンダー

月ごとの不足状況をカレンダー形式で確認できます。

カレンダーでは、日ごとの状態を色分けして表示します。

* 不足なし
* 不足あり
* 大きな不足
* 未設定
* 定休日
* 臨時休業

また、以下のサマリーを表示します。

* 不足ありの日数
* 最大不足人数
* 未設定の日数
* 定休日の日数
* 臨時休業の日数
* 最も注意が必要な日

### 30分単位の不足詳細

必要人数と勤務希望を30分単位で比較し、不足している時間帯を確認できます。

必要人数はまとめた時間帯で登録できますが、不足確認では30分単位に分解して表示します。

例：

必要人数：

* 09:00〜11:00 3人

勤務希望：

* 09:30〜11:00 出勤可能

この場合、09:30以降は出勤希望人数として反映されます。
これにより、入力の手間を増やさずに、細かい時間帯ごとの不足状況を確認できます。

## 工夫した点

### 現場の入力負担を減らす設計

日付ごとの必要人数を毎回手入力するのではなく、曜日ごとの基本パターンを登録し、指定した月へ反映できるようにしました。

また、複数曜日をまとめて選択して登録できるようにし、平日パターンや土日パターンを少ない操作で登録できるようにしています。

### 30分単位での不足確認

勤務希望や必要人数の登録は、開始時刻と終了時刻を指定する形式にしています。
一方で、不足確認では内部的に30分単位へ分解して計算しています。

これにより、入力の手間を増やさずに、実際のシフト作成に近い細かさで不足状況を確認できます。

### 定休日と臨時休業の区別

必要人数が登録されていない日と、営業しない日を区別するため、定休日と臨時休業を別機能として実装しました。

これにより、不足状況カレンダー上で以下を区別できます。

* まだ必要人数が登録されていない日
* 毎週決まって休業する日
* 特定の日だけ休業する日

### データの重複防止

勤務希望、日付別必要人数、曜日パターンについて、時間帯が重複するデータを登録できないようにしています。

これにより、後の不足人数計算で不整合が起きにくいようにしています。

### エラー時の入力保持

入力エラーが発生した場合でも、入力済みの内容が消えないようにしています。
利用者が再入力しやすい画面を意識しました。

### 視覚的に分かりやすい画面

共通CSSを作成し、各画面に統一したデザインを適用しています。

また、削除ボタンは危険操作であることが分かるように赤色で表示し、不足状況や休業日は色分けして表示しています。

## 補足

現在はログイン機能を実装していないため、勤務希望登録時に従業員を選択する仕様としています。
今後、ログイン機能を追加し、ログイン中の従業員に紐づけて勤務希望を登録できるようにする予定です。

また、現在はH2 Databaseを使用して開発しています。
今後、公開環境や本番利用を想定する場合は、永続化可能なデータベースへの変更を検討します。
