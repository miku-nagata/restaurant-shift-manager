# 飲食店向けシフト管理システム

## 概要

飲食店のシフト作成業務を支援するWebアプリケーションです。

従業員情報、勤務希望、必要人数を登録し、必要人数に対して出勤希望人数が足りているかを確認できます。不足している時間帯は赤く表示し、シフト作成時の確認作業を支援します。

## 作成背景

前職の飲食店勤務では、従業員ごとの勤務希望、時間帯ごとの必要人数、扶養内勤務、勤務時間の上限など、シフト作成時に考慮する条件が多くありました。

特に、時間帯ごとの人手不足を確認する作業に負担を感じたため、勤務希望と必要人数を管理し、不足時間を可視化できるシステムを作成しました。

## 主な機能

- 従業員管理
    - 登録
    - 一覧表示
    - 編集
    - 削除
- 勤務希望管理
    - 勤務希望の登録
    - 勤務希望の一覧表示
    - 勤務希望の削除
- 必要人数設定
    - 日付・時間帯ごとの必要人数登録

- 不足時間確認
    - 必要人数と出勤希望人数の比較
    - 不足している時間帯の赤表示

- 必要人数と出勤希望人数を比較し、不足している時間帯を視覚的に分かるようにしました。
- 従業員の時給、月上限時間、月上限金額を登録できるようにし、将来的な労務チェックに拡張しやすい構成にしました。
- Controller、Repository、Entityを分け、Spring Bootの基本的な構成に沿って実装しました。

## 現在実装済みの範囲

### 従業員管理

従業員情報の登録、一覧表示、編集、削除ができます。

登録できる項目は以下のとおりです。

* 氏名
* 雇用形態
* 熟練度
* 時給
* 月上限時間
* 月上限金額

また、氏名や雇用形態、熟練度の未入力チェック、時給や上限値にマイナス値が入力されないようにする入力チェックを実装しています。

### 勤務希望管理

従業員ごとの勤務希望を登録、一覧表示、編集、削除できます。

登録できる項目は以下のとおりです。

* 従業員
* 日付
* 開始時刻
* 終了時刻
* 希望区分

開始時刻が終了時刻以降にならないように入力チェックを行い、エラー時には入力内容が消えないようにしています。

### 必要人数設定

日付・時間帯ごとの必要人数を登録、一覧表示、編集、削除できます。

登録できる項目は以下のとおりです。

* 日付
* 開始時刻
* 終了時刻
* 必要人数

開始時刻と終了時刻の前後チェック、必要人数が1人以上であることのチェックを実装しています。

### 不足時間確認

登録された必要人数と勤務希望を比較し、時間帯ごとの不足人数を確認できます。

不足している時間帯は赤く表示し、状態欄に「不足あり」と表示します。
必要人数を満たしている時間帯は「不足なし」と表示し、不足人数は0として表示します。

これにより、どの時間帯で人員が足りていないかを視覚的に確認できます。

### 画面デザイン

共通CSSを作成し、各画面に統一したデザインを適用しています。

削除ボタンは危険操作であることが分かるように赤色で表示し、不足状態は色付きのバッジで表示しています。
