# 飲食店向けシフト管理システム

## 概要

飲食店のシフト作成業務を支援するWebアプリケーションです。

従業員情報、勤務希望、時間帯ごとの必要人数、定休日、臨時休業を管理し、勤務希望と必要人数を基にシフトを自動作成します。

不足している日や時間帯をカレンダーで可視化し、作成済みシフトは従業員ごとの表やカレンダーで確認できます。

---

## 作成背景

前職の飲食店では、約20名分のシフト作成を担当していました。

シフト作成時には、次のような多くの条件を確認する必要がありました。

- 従業員ごとの勤務希望
- 時間帯ごとの必要人数
- 定休日・臨時休業
- 月間勤務時間や扶養内勤務
- 人手が不足している時間帯

特に、時間帯ごとの不足確認や、曜日ごとの必要人数を繰り返し入力する作業に負担を感じていました。

そこで、入力作業と確認作業を減らし、シフト作成を支援するシステムを制作しました。

## 主な機能

### 従業員管理

従業員情報の登録・一覧表示・編集・削除ができます。

登録項目：

- 氏名
- 雇用形態
- 熟練度
- 時給
- 月間勤務時間上限
- 月間収入上限

### 勤務希望管理

従業員ごとの勤務希望・休み希望を登録できます。

- 登録・一覧表示・編集・削除
- 開始時刻と終了時刻の入力チェック
- 同一従業員・同一日付の重複登録防止
- 30分単位の勤務時間に対応

### 必要人数管理

時間帯ごとに必要な従業員数を設定できます。

- 日付ごとの必要人数登録
- 曜日ごとのパターン登録
- 複数曜日への一括登録
- 指定月への一括反映
- 重複時間帯の登録防止
- 定休日・臨時休業日の除外

### 定休日・臨時休業管理

毎週の定休日と、特定日の臨時休業を登録できます。

不足状況カレンダーでは、未設定日と区別して表示します。

### 不足状況カレンダー

必要人数と勤務人数を30分単位で比較し、月ごとの不足状況を表示します。

表示内容：

- 不足なし
- 不足あり
- 大きな不足
- 未設定
- 定休日
- 臨時休業

不足日数、最大不足人数、未設定日数なども集計して表示します。

### シフト自動作成

勤務希望と必要人数を基に、対象月のシフトを自動作成します。

- 勤務希望の範囲内で割り当て
- 必要人数を満たすことを優先
- 作成済みシフトをデータベースへ保存
- 再作成時は対象月の既存シフトを削除して作り直し

### 作成済みシフト表示

作成したシフトを複数の形式で確認できます。

- 管理者向け作成済みシフト一覧
- 従業員ごとの「自分のシフト」カレンダー
- 全従業員を表示する「みんなのシフト」表

「みんなのシフト」では、日付と曜日を固定し、従業員部分だけを横スクロールできるようにしています。

### ログイン・アクセス制御

- ログイン
- ログアウト
- 未ログイン時のアクセス制限
- パスワード表示切替

## 工夫した点

### 現場経験を基にした画面設計

前職で実際にシフト作成を担当した経験から、入力作業の削減と不足状況の見つけやすさを重視しました。

### 必要人数パターンの一括反映

曜日ごとの必要人数を登録し、指定した月へ一括反映できるようにしました。

定休日や臨時休業日は自動的に除外します。

### 30分単位の不足確認

登録は時間帯単位で行い、内部では30分単位に分割して必要人数と勤務人数を比較しています。

入力のしやすさと、細かな不足確認を両立しました。

### 日付と従業員からシフトを検索

作成済みシフトは、日付と従業員IDをキーとするMapに整理し、Thymeleafで表形式に表示しています。

### スマートフォンでの閲覧対応

「みんなのシフト」では、日付・曜日列と従業員名を固定し、シフト部分だけをスクロールできるようにしました。

### 入力エラーへの対応

- 重複時間帯の登録防止
- 入力値のチェック
- エラー発生時の入力内容保持
