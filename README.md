# 飲食店向けシフト管理システム

## 概要

飲食店のシフト作成業務を支援するWebアプリケーションです。

従業員情報、勤務希望、必要人数を登録し、必要人数に対して出勤希望人数が足りているかを確認できます。不足している時間帯は赤く表示し、シフト作成時の確認作業を支援します。

## 作成背景

前職の飲食店勤務では、従業員ごとの勤務希望、時間帯ごとの必要人数、扶養内勤務、勤務時間の上限など、シフト作成時に考慮する条件が多くありました。

特に、時間帯ごとの人手不足を確認する作業に負担を感じたため、勤務希望と必要人数を管理し、不足時間を可視化できるシステムを作成しました。

## 主な機能

- 従業員管理

    - 一覧表示

    - 勤務希望の一覧表示

## 現在実装済みの範囲

### 従業員管理

従業員情報の登録、一覧表示、編集、削除ができます。

登録できる項目は以下のとおりです。

* 氏名
* 雇用形態
* 熟練度
* 時給
* 月上限時間
* 月上限金額

氏名、雇用形態、熟練度の未入力チェックを実装しています。
また、時給、月上限時間、月上限金額にマイナス値が入力されないようにしています。

### 勤務希望管理

従業員ごとの勤務希望を登録、一覧表示、編集、削除できます。

登録できる項目は以下のとおりです。

* 従業員
* 日付
* 開始時刻
* 終了時刻
* 希望区分

開始時刻が終了時刻以降にならないように入力チェックを行っています。
また、同じ従業員が同じ日付・重なる時間帯に勤務希望を二重登録できないようにしています。

エラーが発生した場合でも、入力内容が消えないようにし、再入力の負担を減らしています。

### 必要人数設定

日付・時間帯ごとの必要人数を登録、一覧表示、編集、削除できます。

登録できる項目は以下のとおりです。

* 日付
* 開始時刻
* 終了時刻
* 必要人数

開始時刻と終了時刻の前後チェック、必要人数が1人以上であることのチェックを実装しています。
また、同じ日付・重なる時間帯に必要人数設定を二重登録できないようにしています。

### 不足時間確認

登録された必要人数と勤務希望を比較し、時間帯ごとの不足人数を確認できます。

必要人数設定はまとめた時間帯で登録できますが、不足確認画面では30分単位に分解して表示します。
これにより、例えば「09:00〜11:00に3人必要」という設定に対して、勤務希望が「09:30〜11:00」の場合でも、09:30以降の時間帯では出勤希望人数として反映できます。

不足している時間帯は赤く表示し、状態欄に「不足あり」と表示します。
必要人数を満たしている時間帯は「不足なし」と表示し、不足人数は0として表示します。

### 画面デザイン

共通CSSを作成し、各画面に統一したデザインを適用しています。

削除ボタンは危険操作であることが分かるように赤色で表示しています。
また、不足状態は色付きのバッジで表示し、視覚的に分かりやすくしています。

## 工夫した点

飲食店のシフト作成業務を想定し、現場で起こりやすい入力ミスや確認漏れを防ぐことを意識しました。

勤務希望や必要人数は、開始時刻と終了時刻を指定してまとめて登録できるようにしています。
一方で、不足確認では30分単位に分解して計算することで、入力の手間を増やさずに細かい時間帯ごとの不足状況を確認できるようにしました。

また、同じ従業員の重複した勤務希望や、同じ時間帯の必要人数設定が登録されないようにし、データの整合性を保てるようにしています。

エラー時にも入力内容が保持されるようにし、利用者が再入力しやすい画面を意識しました。
